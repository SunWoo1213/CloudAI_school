# 5주차: 모던 투두 리스트 (Modern To-Do List)

이 프로젝트는 HTML, CSS, JavaScript를 활용하여 구현된 직관적이고 모던한 스타일의 To-Do 리스트 웹 애플리케이션입니다. 검색, 마감일 관리, 동적 상태 변화(임박/지연), 그리고 상세한 콘솔 로깅 기능을 포함합니다.

## 디렉토리(파일) 구조

```text
5주차/
├── index.html   # 메인 HTML 마크업 및 JavaScript 로직이 포함된 진입점
├── style.css    # 투두 리스트의 전체 디자인, 색상, 애니메이션을 담당하는 스타일시트
└── README.md    # 현재 가이드라인 문서
```

## 주요 기능 요약

1. **할 일 상태 및 마감일(Due Date) 관리 추가**
   * 추가 폼에 `<input type="datetime-local">`을 도입하여 구체적인 마감 시간을 설정할 수 있습니다.
   * 입력값이 공백일 경우 데이터 오염을 막기 위한 방어 코드(알림)가 구현되어 있습니다.
   
2. **할 일 내용 기반 실시간 검색**
   * 상단에 위치한 검색창에 텍스트를 입력하면 목록이 즉시(oninput 이벤트) 필터링됩니다.

3. **마감 상태 시각화 (동적 CSS 적용)**
   * **기본(Normal)**: 마감일 24시간 이상
   * **임박(Warning)**: 마감일 24시간 미만 (노란색 톤 배경 및 테두리 `.warning` 클래스)
   * **지연(Overdue)**: 마감일 초과 (붉은색 톤 배경 및 진한 텍스트 `.overdue` 클래스)
   * **실시간 갱신**: `setInterval`을 이용해 1분 주기(60000ms)마다 상태를 체크하여 화면을 리렌더링합니다. (※ 단, 완료 표기가 된 항목은 상태를 계산하지 않고 우선적으로 취소선을 표시합니다.)

4. **콘솔 로깅(Console Logging) 기능**
   * 데이터 추가, 상태 변경(완료/미완료 토글), 삭제가 발생할 때마다, 현재 어떤 항목이 어떻게 대상이 되었고 남은 미완료 항목이 총 몇 개인지 개발자 도구의 콘솔 화면에 즉시 로깅됩니다.

## 코드 소스 작성 밎 수정 가이드라인

향후 코드를 수정하거나 기능을 확장하실 때 다음의 가이드라인과 룰을 준수해 주세요.

### 1. HTML (index.html) 가이드라인
* **의미론적 태그 사용**: 목록 표시는 `<ul>` 및 `<li>`를 유지하고 폼 관련 요소는 명확한 `id`와 부모 역할인 `class` 컨테이너로 감싸 구조를 잘 분리합니다.
* **스타일 분리 원칙**: 인라인 CSS(`style=""`) 사용을 지양합니다. 스타일은 항상 `style.css`에서 클래스나 ID 선택자를 통해 정의해야 합니다.

### 2. CSS (style.css) 가이드라인
* **가독성과 미학 확보(Modern UI)**: 모던한 화면을 위해 `Inter` 폰트와 파스텔 톤 그라데이션, 그리고 명확한 마이크로 애니메이션(`transform`, `transition`)을 지속적으로 활용하세요.
* **상태 클래스 규칙**: `.warning`, `.overdue`, `.completed` 등 **상태를 나타내는 클래스**는 항상 가장 높은 우선순위로 화면에 피드백을 주어야 하므로 변경 시 로직과 색상 팔레트 규칙(노랑, 빨강, 파랑/보라 계열 등)을 통일성 있게 유지합니다.

### 3. JavaScript (index.html 내 script) 가이드라인
* **단일 상태 관리 (`todos` 배열)**:
  * DOM(문서)을 직접 조작해 데이터를 뽑아 쓰지 않습니다.
  * 추가, 토글, 삭제 등의 액션은 반드시 전역 상태인 `todos` 빈 배열 데이터를 기반으로 가공한 후, 무조건 **`render()`** 함수를 통해 DOM에 일괄 반영해야 데이터 동기화 이슈를 방지할 수 있습니다.
* **이벤트 처리 추상화**:
  * 추가 관리는 `addTodo`, 토글 관리는 `toggleTodo`, 삭제 관리는 `deleteTodo` 등 함수로 세분화되어 있습니다. 향후 기능을 더 추가할 때도 목적에 맞는 단일 책임을 갖는 새 함수를 만들어 연결하는 방식을 추천합니다.
* **로깅 및 예외 관리**: 
  * 비즈니스 로직(예: 데이터 삭제 결합이나 오류 발생 예상 지점)에는 항상 예방 코드(`null`, `undefined`, 빈 문자열 방어 등)와 디버깅을 돕는 `console.log()` 흔적을 남겨주세요.
