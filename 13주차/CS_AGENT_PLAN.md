# 13주차 — 대화형 CS 처리 에이전트 지침서

## 1. 목표

12주차에서 만든 고객 서비스 에이전트(`12주차/customer_service.py`)는 미리 정의된
3개의 데모 발화(`DEMO_TURNS`)를 순차 실행하는 **자동 실행형**이었다.

13주차의 목표는 동일한 에이전트 구조를 유지하되, 사용자가 **터미널에서 직접
입력**하면서 대화를 이어가는 **대화형(interactive) REPL** 구조로 바꾸는 것이다.

핵심:

- 통신 패턴(Orchestrator-Worker, "Agent as Tool")은 그대로 유지한다.
- 고객 서비스 에이전트가 1차 응대 창구이고, 기술 문의는 `call_agent` 도구를 통해
  `tech_support` 서브에이전트로 위임된다.
- 대화 맥락(`Memory`)은 세션 동안 유지되어, 이전 발화에서 언급된 주문번호·SDK
  버전·오류 코드 등이 후속 턴에서 자연스럽게 참조되어야 한다.
- 사용자는 종료 명령(`/exit`, `/quit`, `:q`)으로 깔끔하게 빠져나갈 수 있어야
  한다.

## 4. 13주차 폴더 구조

```
13주차/
├── CS_AGENT_PLAN.md        # 본 지침서 (현재 파일)
└── customer_service_cli.py # 대화형 CLI 진입점 + 에이전트 로직
```

## 5. 대화형 REPL 사양

### 5.1 시작 화면

프로그램이 시작되면 다음을 1회 출력한다.

```
==================================================================
  대화형 CS 처리 에이전트 (13주차)
  - 일반 문의는 고객 서비스 에이전트가 직접 답변합니다.
  - 기술 문의는 자동으로 tech_support 에이전트로 위임됩니다.
  - 종료하려면 /exit, /quit, :q 중 하나를 입력하세요.
==================================================================
```

### 5.2 입력 프롬프트

매 턴마다 다음 형식의 프롬프트를 출력한다.

```
고객(나) > _
```

빈 입력(공백·엔터만 친 경우)은 무시하고 다시 프롬프트를 띄운다.

### 5.3 종료 처리

- `/exit`, `/quit`, `:q` 중 하나가 입력되면 종료 메시지를 출력하고 정상 종료한다.
- `Ctrl-C`(`KeyboardInterrupt`) 또는 EOF(`Ctrl-Z` + Enter, `EOFError`)도
  동일하게 종료 처리한다.
- 종료 시 누적 대화 턴 수를 간단히 안내한다. 예: `세션을 종료합니다. (총 5턴)`

### 5.4 응답 출력

매 턴 응답은 다음 두 정보를 출력한다.

1. **위임 여부 표시(선택적, 한 줄)**: 이번 턴에 `call_agent` 가 호출되었다면
   `[위임됨 → tech_support]` 한 줄을 응답 위에 출력. 호출되지 않았다면 생략.
2. **최종 답변**: `상담원 > <에이전트 응답 텍스트>` 형식.

12주차의 `_print_separator` 같은 굵은 구분선은 매 턴 출력하지 않는다. 대화 흐름이
끊겨 보이지 않게 단순한 줄바꿈만 사용한다.

### 5.5 에러 처리

- `OPENAI_API_KEY` 미설정 → 시작 시 명확한 안내 문구 출력 후 종료(exit code 1).
- 단일 턴에서 OpenAI 호출 실패(`Exception`) → 해당 턴 응답으로
  `[오류] {예외 메시지}` 를 출력하고 REPL 은 계속한다. 한 번의 API 오류로 세션
  전체를 죽이지 않는다.
- `call_agent` 가 `success: False` 를 돌려준 경우는 12주차와 동일하게 도구 결과
  메시지로 메모리에 남고, 고객 서비스 에이전트가 자연스럽게 사과·재안내한다.

## 6. 구현 단계

### Step 1. 12주차 코드 이식

`12주차/customer_service.py` 의 모듈 상단~`run_customer_service_agent` 까지를
`13주차/customer_service_cli.py` 로 복사한다. 단, 다음만 조정한다.

- `ENV_FILE_PATHS` 에 `Path(__file__).with_name(".env")` 가 13주차 폴더를
  가리키도록 자동 확장된다(파일 위치만 옮기면 됨).
- `MODEL_NAME` 은 12주차와 동일하게 `"gpt-4.1-mini"` 유지.

### Step 2. REPL 루프 작성

`main()` 을 새로 작성한다. 시그니처는 동일하게 인자 없음. 동작:

1. `load_env_files()` 호출.
2. `OPENAI_API_KEY` 미설정 시 안내 후 `sys.exit(1)`.
3. `AgentRegistry` 생성, `tech_support` 등록.
4. `Memory` 1개 생성(세션 전체 동안 공유).
5. 시작 화면 출력.
6. 무한 루프:
   - `input("고객(나) > ")` 으로 한 줄 받기.
   - 빈 입력 → 계속.
   - 종료 명령 → break.
   - 그 외 → `run_customer_service_agent(...)` 호출. `before_len` 로 신규
     아이템만 추려서 위임 여부 검사 → 응답 출력.
   - `KeyboardInterrupt` / `EOFError` → break.
   - 그 외 `Exception` → `[오류] ...` 출력 후 계속.
7. 종료 메시지 출력.

### Step 3. 수동 테스트 시나리오

12주차의 자동 검증을 사용자가 수동으로 재현하면 된다. 예시 대화:

```
고객(나) > 안녕하세요. 어제 주문번호 ORD-9988 로 결제했는데 배송 며칠 걸리나요?
상담원 > ...(일반 응대, 위임 없음)...

고객(나) > 그 주문은 PaymentSDK 2.3.0 입니다.
상담원 > ...(일반 응대, 위임 없음)...

고객(나) > 그 SDK 로 결제 API 호출하면 401 Unauthorized 가 뜹니다.
[위임됨 → tech_support]
상담원 > ...(ORD-9988 / PaymentSDK 2.3.0 / 401 같은 키워드가 인용된 기술 답변)...

고객(나) > /exit
세션을 종료합니다. (총 3턴)
```

기대 결과:

- 1·2번째 발화에서 `[위임됨]` 표시가 나타나지 **않는다**.
- 3번째 발화에서 `[위임됨]` 표시가 나타나고, 답변에 이전 맥락 키워드가 1개 이상
  포함된다.

### Step 4. 실행 방법

프로젝트 루트의 가상환경을 활성화한 뒤 실행:

```powershell
.\.venv\Scripts\Activate.ps1
python .\13주차\customer_service_cli.py
```

또는 uv 환경이라면:

```powershell
uv run python .\13주차\customer_service_cli.py
```

## 7. 비목표(Non-goals)

- 멀티 사용자 동시 세션, 세션 영속화(DB 저장) — 단일 세션 인메모리만.
- 자동 회귀 테스트 스크립트 — 12주차에서 이미 수행했으므로 13주차는 수동 확인.
- 기술 지원 외 다른 전문 에이전트(환불 전담, 영업 등) 추가 — 추후 과제.
- 스트리밍 응답, 색상 출력, `rich` 사용 — 표준 `print` 만 사용.

## 8. 완료 기준(Definition of Done)

1. `13주차/customer_service_cli.py` 가 단독으로 실행 가능하다.
2. 빈 입력 / `/exit` / `Ctrl-C` 가 의도대로 동작한다.
3. 위 Step 3 시나리오에서 기술 문의 턴에 `[위임됨 → tech_support]` 가 1회 출력된다.
4. 기술 에이전트 답변이 이전 턴의 맥락 키워드(주문번호·SDK 버전·오류 코드 중
   최소 1개)를 인용한다.
5. 12주차 코드의 핵심 구조(Memory / AgentRegistry / call_agent /
   run_customer_service_agent / run_tech_support_agent)가 보존되어 있다.
